# AI 全域营销可演示闭环设计

## 1. 目标与范围

本迭代将现有创意生成、资产审核、Campaign 工作台、会场编辑器和智能体展示串联为一条可操作的端到端链路：

> 创建活动 → 生成 4 案 → 选择与精修 → 审核归档 → 智能体编排 → 人工批准投放 → 多触点上线 → 实时数据回收 → AI 实验建议 → 扩量或回滚 → 结果复盘

本轮使用前端领域状态机和模拟事件流，不接入真实生图、素材存储、商品权益、渠道发布或数据平台。模拟服务必须遵守未来真实接口的领域合同，避免 UI 与演示脚本耦合。

## 2. 产品成功标准

- 用户从首页开始，能够连续完成完整链路，无需刷新页面或依赖口头讲解。
- 每个阶段都显示当前状态、执行主体、产出、证据和明确的下一步操作。
- 资产、投放、实验和指标均可追溯到同一个 Campaign ID。
- 演示包含成功链路及至少一个可恢复异常：创意失败重试或渠道部分失败。
- 数据异常能够触发 AI 建议；实验可根据护栏进入扩量或回滚结局。
- 页面刷新后可从本地快照恢复当前 Campaign；可一键重置演示。

## 3. 核心领域模型

### 3.1 DemoCampaign

包含 Campaign ID、Brief、当前阶段、整体状态、选中资产、会场版本、触点、智能体任务、指标快照、实验、事件日志和更新时间。

### 3.2 阶段状态机

```text
brief
  → generating
  → asset_review
  → orchestration
  → launch_review
  → publishing
  → live
  → optimizing
  → completed
```

任一线上阶段可进入 `paused`；发布或实验触发护栏时进入 `rolling_back`，完成后回到最近一个稳定版本。阶段只能通过领域动作迁移，页面不得直接改写状态。

### 3.3 Action 与 Event

领域动作包括创建 Campaign、保存资产、提交审核、批准投放、发布触点、记录指标、批准实验、扩大流量、暂停和回滚。每个动作产生一条不可变事件，包含时间、执行主体、输入、结果、证据和关联对象。

## 4. 页面与交互设计

### 4.1 创作首页

- 提交 Prompt 时同时创建 DemoCampaign，而非只生成孤立 Brief。
- 顶部显示当前活动状态；存在未完成活动时提供“继续活动”入口。
- 保留快速生成与全链路活动双入口，两者均可升级为 Campaign。

### 4.2 创意资产生产

- 沿用 4 个独立候选任务、失败重试、规则检查、文案精修和审核归档。
- 审核通过后显示“加入 Campaign 并继续”，将选中资产写入统一状态。
- 资产版本、压板版本和规则证据进入事件日志。

### 4.3 Campaign 指挥中心

- 顶部增加阶段进度条：创意、会场、审批、投放、监控、优化。
- 营销活动智能体按顺序生成和执行专业智能体任务，并展示任务状态、证据及耗时。
- “提交投放审批”打开摘要，展示触点、资产、会场版本、预算护栏和风险检查。
- 批准后模拟多触点发布；其中一个触点首次发布失败，可单独重试并成功。

### 4.4 实时数据与优化

- 活动上线后按固定时间步推进指标：曝光、CTR、到达率、加购率、CVR、GMV。
- 第一个监控窗口模拟会场到达率下降，数据分析智能体生成带置信度和证据的异常结论。
- 营销活动智能体提出“会场 B 版本 10% 流量实验”，需人工批准。
- 用户可选择“胜出扩量”或“触发护栏”演示分支；默认推荐胜出扩量。
- 扩量分支更新会场权重和业务结果；护栏分支展示自动停流、回滚和稳定版本恢复。

### 4.5 全链路时间线

- Campaign 工作台提供统一时间线，聚合用户、营销活动智能体、专业智能体和规则引擎事件。
- 每条事件显示执行者、动作、对象、结果、证据和时间。
- 支持查看当前状态，不增加复杂筛选功能。

## 5. 架构边界

```text
React 页面
   ↓ 只派发 Action
Campaign Store / Reducer
   ↓ 领域状态迁移
Demo Runtime
   ├─ Asset Generator Simulator
   ├─ Agent Orchestrator Simulator
   ├─ Channel Publisher Simulator
   └─ Metrics Stream Simulator
   ↓ 生成 Event
Event Timeline + Local Snapshot
```

- UI 不直接维护重复 Campaign 状态。
- 模拟器通过显式命令推进，不使用不可控的无限定时任务。
- 运行中的短定时器在页面卸载时清理。
- Local Storage 只保存可序列化快照；Schema 版本不匹配时安全重置。

## 6. 异常与安全设计

- 创意生成失败：只重试单个候选，不重置批次。
- 渠道部分失败：已成功触点保持上线，失败触点可幂等重试。
- 数据延迟：不产生判胜结论，自动化降级为等待人工观察。
- 实验护栏触发：停止实验流量并恢复稳定会场版本。
- 重复点击：同一动作在运行中禁用，避免重复事件和重复发布。
- 刷新恢复：若快照损坏或版本不兼容，提示后回到安全初始状态。

## 7. 测试与验收

### 7.1 领域测试

- 合法和非法阶段迁移。
- 资产审核后升级为 Campaign 资产。
- 多智能体任务依赖及完成顺序。
- 渠道部分失败、幂等重试和全量发布完成。
- 指标异常、实验批准、扩量和护栏回滚。
- 快照保存、恢复和版本失配。

### 7.2 浏览器验收路径

1. 编辑 Prompt 并创建 Campaign。
2. 等待 4 案生成，重试失败方案，选择并精修资产。
3. 提交审核并加入 Campaign。
4. 查看智能体任务执行，提交并批准投放。
5. 重试失败触点，确认活动上线。
6. 推进监控窗口，查看 AI 异常结论。
7. 批准 10% 实验，分别验证扩量与回滚分支。
8. 刷新页面确认进度恢复，再执行一键重置。

### 7.3 工程验收

- 单元测试、类型检查和生产构建全部通过。
- 1440px 桌面视口无横向溢出，关键动作可键盘完成。
- 应用控制台无错误，所有异步任务可终止和恢复。

## 8. 明确不包含

- 真实模型推理、生图或素材上传。
- 真实商品、权益、预算、用户、人群和渠道接口。
- 服务端数据库、登录、多人协作和权限系统。
- 真实订单归因、线上流量操作和生产环境发布。

这些能力在演示闭环稳定后，依据相同领域合同逐个替换模拟器。
