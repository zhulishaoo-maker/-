# 创意资产生产闭环设计

## 目标

在现有“输入 Brief → 固定生成 4 张候选”的基础上，补齐可演示的创意生产、规则检查、单图重试、方案精修、审核与归档闭环。此迭代使用本地状态模拟异步任务，不接入真实生图、存储或审核服务。

## 产品流程

1. 运营提交结构化 Brief，系统创建一批 4 个候选任务。
2. 候选按独立状态推进：排队、生成、规则检查、就绪或失败。
3. 底图由生成任务负责；京东大促品牌压板、搜索框、CTA 和安全区由确定性规则层合成并校验。
4. 运营选择候选，在检查台查看品牌、文案、安全区和尺寸结果。
5. 失败候选可单独重试，不影响其余方案；就绪候选可进入精修。
6. 精修后提交审核，审核通过即形成带版本号的归档资产，并可升级为 Campaign 资产。

## 界面设计

- 顶部增加批次状态与“4 个候选”的生产进度，明确当前阶段。
- 候选卡展示独立状态、版本和规则分数；失败态提供就地重试。
- 选中方案后打开检查台，集中展示确定性压板、搜索框、安全区、文案长度和尺寸规则。
- 精修采用轻量侧边面板，仅开放主标题、副标题和 CTA；品牌压板保持锁定。
- 底部操作区随状态变化，支持重试、精修、提交审核和归档结果反馈。

## 状态与边界

- `CandidateStatus`: queued / generating / validating / ready / failed / refining / reviewing / approved。
- 每张候选独立保存状态、进度、规则结果、版本和可编辑文案。
- 批次状态由 4 张候选派生，不维护第二份可变状态。
- 异步模拟必须在组件卸载时清理；重复点击不得创建并发重试或重复审核。
- 品牌压板字段只读；规则未通过时禁止提交审核。

## 测试与验收

- 领域测试覆盖批次创建、状态迁移、规则聚合和重试。
- 组件流程覆盖 4 张候选独立状态、失败重试、精修和审核成功。
- 类型检查、单测、生产构建与浏览器控制台全部通过。
- 1440px 桌面视口无横向溢出，核心操作可用键盘完成。

## 后续衔接

归档资产保留 Campaign、Brief、候选、压板版本和审核状态标识，为营销活动智能体委派设计任务、渠道分发和数据归因提供稳定合同。
