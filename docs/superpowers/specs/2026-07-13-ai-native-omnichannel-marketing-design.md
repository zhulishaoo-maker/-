# AI Native 全域营销自动化平台设计

## 1. 产品定位

本项目从“AI 会场生成与运营”升级为“AI Native 全域营销自动化平台”。平台以营销活动 `Campaign` 为统一运营单元，支持快速生成单项资产，也支持围绕用户旅程一次性规划、生产、审核、投放和优化开屏、传播海报、Banner/站内资源位和营销会场。

核心闭环：

```text
活动意图 → 旅程策略 → 多资源生产 → 规则校验 → 审核投放
    ↑                                                     ↓
策略记忆 ← AI 实验与优化 ← 跨触点数据与转化归因
```

## 2. 产品原则

- AI 负责理解意图、制定计划、生成创意、提出实验和复盘。
- 确定性系统负责品牌压板、搜索框压板、尺寸、安全区、权益、权限、发布和回滚。
- 一个 Campaign 内的所有资产共享主题、权益、人群、商品、主视觉、目标和数据标识。
- 快速生成的单项资产可随时升级为 Campaign 资产，进入审核、投放和运营闭环。
- 每次生成固定输出 4 张候选，不向用户暴露生成数量设置。
- 所有自动优化都必须有证据、权限、流量边界、停止条件和回滚记录。

## 3. 产品入口

### 3.1 快速生成

面向明确的单项任务，首版包含：

- 自由模式。
- 开屏。
- 传播海报。
- 营销会场。
- 图片处理。
- 批量拓展/多尺寸适配。
- 京东站内 Banner 和资源位。

快速模式优先缩短从意图到首版资产的时间，不要求用户先创建完整 Campaign。

### 3.2 全链路活动

运营一次性定义主题、权益、人群、商品池、周期、经营目标和护栏。Campaign Agent 产出用户旅程、资源组合、创意策略、会场策略、实验计划和投放计划。

## 4. 句内结构化 Prompt Composer

### 4.1 交互模型

输入框展示一段可直接编辑的自然语言默认提示词。品牌压板、搜索框压板、风格、尺寸、权益、商品等配置不作为框外独立表单，而是作为句子中的可编辑下拉槽位。

开屏默认句式示例：

> 为【夏日美妆狂欢】生成【开屏】创意，使用【京东大促品牌压板】和【京东底部商品词搜索框】，搜索词为【夏日防晒新品】，视觉采用【冰感未来】风格，尺寸为【3:4·750×1000】。

用户可直接改写整段文字，也可点击句内槽位选择已注册的标准值。提示词文本和槽位值在提交时合并为结构化 `GenerationBrief`。

### 4.2 任务模板

- 开屏：京东大促品牌压板、京东搜索框压板、搜索词、风格、安全区和尺寸。
- 传播海报：京东品牌压板、二维码、分享文案、风格和比例。
- Banner/资源位：京东资源位模板、标题、CTA、品牌压板、安全区和尺寸。
- 营销会场：权益、人群、商品池、风格、页面大纲和经营目标。

### 4.3 京东大促品牌压板

京东大促品牌压板是经过审核和版本化的确定性模板，包含：

- 京东 Logo、大促标识和品牌色。
- 主标题、权益和商品信息的层级与位置。
- 法务脚注、活动规则和禁放区。
- Logo、CTA、搜索框和文字的安全区。
- 必须项、可变项和禁用项。

### 4.4 京东搜索框压板

搜索框是最终素材中的固定压板组件，不是用于查找参考图的工具。模板包含：

- 搜索框外观、边框、圆角和图标。
- 搜索词和 CTA 的排版。
- 顶部或底部位置、安全区和禁放区。
- 品牌词、商品词等已注册变体。

## 5. 创意生产链路

1. Prompt Composer 产出结构化 `GenerationBrief`。
2. AI 根据参考 KV、商品、风格和尺寸生成 4 张无压板底图。
3. 生成过程根据所选品牌和搜索框模板预留安全区。
4. 确定性合成引擎叠加 Logo、大促标识、权益、脚注、搜索框、搜索词和 CTA。
5. 校验引擎检查尺寸、安全区、价格权益、文案、品牌一致性和素材授权。
6. 通过校验的 4 张候选进入审核、投放或升级为 Campaign 资产。

AI 可生成背景、人物、商品摆放和创意构图，但不直接绘制 Logo、大促标识、固定权益、法务脚注、搜索框和 CTA。

## 6. Campaign Agent 与用户旅程

Campaign Agent 将活动拆解为五个可组合阶段：

1. 曝光：开屏、高曝光 Banner 和首页资源位。
2. 兴趣：品类 Banner、商品资源位和内容卡片。
3. 到达：营销会场、品类楼层和个性化承接版本。
4. 转化：权益领取、加购、下单和支付。
5. 再传播：分享海报和动态商品信息。

活动工作台以旅程图组织资产、资源位、会场版本、投放状态、指标和 AI 建议。

## 7. 系统架构

### 7.1 智能体控制面

- 理解 Campaign Brief 和单项 Generation Brief。
- 规划用户旅程、资源组合、人群和实验。
- 调用创意生成、图片处理、会场编排、压板、校验、发布和数据工具。
- 维护品牌规范、运营偏好、历史活动和实验结论。
- 输出可解释的生产和调权决策。

### 7.2 创意与资产工厂

- 文案生成、图像生成、抠图、扩图、替换和修复。
- 京东大促品牌压板库、京东搜索框压板库和资源位模板库。
- 多尺寸批量适配、安全区检测和确定性合成。
- 资产版本、衍生关系、审核状态和归档。

### 7.3 会场与渠道执行面

- 受控会场组件、Page Schema、选品和页面编排。
- 京东站内资源位规范、排期、频控、审核和投放适配器。
- 不可变发布版本、灰度、实验、停流和回滚。
- 权益、价格、库存、素材授权、预算和投放规则校验。

### 7.4 实时反馈与实验面

- 曝光、点击、互动、到达、加购、下单、支付和分享事件。
- Campaign、Journey Stage、Placement、Creative 和 Venue Version 统一数据标识。
- 对照分流、最小样本、置信度、最小业务效应量和护栏。
- 数据延迟、埋点缺失、频控、预算、投诉和系统健康监控。

## 8. 跨触点数据与归因

统一追踪链路：

```text
Campaign → Journey Stage → Placement → Creative Variant → Venue Version → Order
```

系统辅助识别：

- 素材问题：曝光充足、点击率偏低。
- 资源位问题：创意点击表现正常、有效曝光不足或人群不匹配。
- 承接问题：点击正常、会场到达、加购或支付偏低。

MVP 使用同期对照和组合实验评估增量。多触点归因用于诊断和辅助决策，不单独作为自动扩量的因果依据。

## 9. 组合实验

实验单元 `ExperienceVariant` 将创意、资源位、人群和会场版本组合为一个完整用户体验。例如：

```text
新客控油开屏 B → 新客券资源位 C → 控油首楼会场 B
```

该方式避免局部素材高点击但整体转化下降。每次实验优先只改变一类主变量；组合实验需明确主变量、关联不变项和停止条件。

## 10. 分级自治

- L0 生产：AI 生成资产和会场，人工审核后投放。
- L1 建议：AI 识别素材、资源位或承接问题，提交跨触点优化建议。
- L2 实验：AI 在已批准的资源位、人群和流量上限内启动实验。
- L3 自治：已验证的组合策略可在护栏内自动调权，异常时立即停流和回滚。

可在护栏内自动：

- 已审核创意之间的切换和权重调整。
- 人群、时段和低风险资源位的小流量实验。
- 会场商品排序和非核心楼层顺序。

始终需要人工审批：

- 新主视觉、新核心文案、权益、价格和交易规则。
- 新渠道、新资源位和全量流量切换。
- 预算上限、频控规则和整体用户旅程的改变。

## 11. 核心数据对象

- `Campaign`：活动身份、目标、周期、责任人、状态和自治等级。
- `CampaignBrief`：主题、权益、人群、商品池、预算、目标权重和护栏。
- `GenerationBrief`：任务类型、自然语言、句内槽位、输入资产和生成约束。
- `JourneyStage`：曝光、兴趣、到达、转化或再传播阶段及对应目标。
- `MasterCreative`：主视觉、核心文案、品牌约束和权益。
- `OverlayTemplate`：京东大促品牌压板、搜索框压板或资源位模板的版本化定义。
- `AssetVariant`：渠道、尺寸、人群、风格、底图、压板成品和审核状态。
- `Placement`：资源位规范、排期、流量、人群、频控和投放状态。
- `VenueVersion`：会场 Page Schema、内容引用和不可变发布版本。
- `ExperienceVariant`：创意、资源位、人群和会场版本的实验组合。
- `Decision`、`MetricSnapshot`、`GuardrailEvent`：AI 决策、指标快照和护栏事件。

## 12. 异常与降级

- 生成失败：保留已完成的候选和资产，对失败任务单项重试。
- 压板、尺寸、安全区、权益或授权不合规：阻止对应资产进入投放。
- 渠道或资源位发布失败：保留已投放旧版，不影响其他触点。
- 埋点缺失、数据延迟或链路标识断裂：暂停跨触点归因和自动优化。
- 预算、频控、投诉、转化或系统健康越界：停止实验新流量并回滚。
- 创意与会场的权益、人群或商品表达不一致：组合实验不得发布。

## 13. MVP 范围

MVP 包含：

- 首页快速生成与全链路活动双入口。
- 句内结构化 Prompt Composer 和任务默认句式。
- 开屏、海报、Banner/站内资源位、营销会场和批量多尺寸。
- 京东大促品牌压板、京东搜索框压板和资源位模板。
- 每次固定 4 张候选的生产、校验、预览和审核。
- Campaign、用户旅程、资产树、资源位、会场版本和投放状态。
- 统一数据标识、指标看板、对照分流和组合实验。
- L0–L3 自治等级、审批、灰度、停流和回滚。

MVP 不包含：

- Push、短信和站外广告投放。
- AI 直接绘制 Logo、搜索框或法务脚注。
- AI 自动修改价格、权益、预算上限和交易规则。
- 无对照、无样本门槛或无回滚方案的全量自动调整。

## 14. 测试策略

- Prompt 文本与句内槽位同步、默认值、缺失项和任务模板测试。
- 京东大促品牌压板、搜索框压板、资源位模板版本与合成测试。
- 尺寸、安全区、禁放区、文案溢出、权益和授权规则测试。
- 四张候选完整性、单项重试、资产衍生关系和版本测试。
- Campaign、Journey、Placement、Creative、Venue 和 Order 标识串联测试。
- 实验分流、数据质量、样本门槛、护栏和自动回滚测试。
- 创意、资源位和会场组合一致性与端到端验收。

## 15. MVP 验收标准

- 单项任务从提交到首批 4 张候选的中位时间不超过 5 分钟。
- 一个 Campaign 可一次生成至少 4 类资源和一个完整会场。
- 多尺寸与压板成品通过模板、安全区和结构校验的比例达到 95%。
- 100% 投放资产可追溯至 Campaign、Generation Brief、底图、压板版本、审核和投放记录。
- 会场及营销资产生产的人工工时相对现有流程中位数降低至少 70%。
- 异常资源位可独立停止和回滚，不影响 Campaign 的其他触点。
- 至少完成一类“创意＋资源位＋会场”组合的预注册同期对照实验，并达到事先设定的最小业务效应量。
- 100% 自动调权包含证据、自治权限、流量范围、停止条件和回滚记录。
