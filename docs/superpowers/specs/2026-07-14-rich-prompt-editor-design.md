# 整段可编辑 Prompt 设计

## 目标

将当前“固定句式＋可编辑灰色槽位”升级为整段可编辑的富文本 Prompt。运营点击白色输入区内任意普通文字即可修改句式；活动主题、品类、权益、压板、风格和尺寸仍以内嵌灰色组合框呈现；任务类型继续由下方任务卡控制。

## 方案

采用分段富文本编辑器。普通句式被拆为九个可编辑文本段，参数槽位是不可被富文本操作破坏的独立组合框。相比单个 textarea，该方案能保留清晰的结构化参数；相比无约束 `contenteditable` 根节点，它能避免用户误删槽位或产生不可解析的 HTML。

## 数据模型

`ComposerState` 增加 `promptCopy`，保存九段普通文字。`composePrompt` 按“文本段—槽位—文本段”的稳定顺序生成最终 Prompt。用户看到的句子是提交的唯一文本来源，不再由固定字符串重新生成。

## 交互

- 点击普通文字直接出现光标并编辑，可换行显示但提交时归一化多余空白。
- 普通文字为空时允许保留，支持用户删掉不需要的连接词。
- 灰色槽位仍支持自由输入、推荐筛选及键盘选择。
- 红色任务类型为结构化节点，不允许在正文中误删。
- 编辑时显示轻量聚焦背景和“文字可编辑”提示，不增加工具栏。

## 验收

- 九段普通文字和七个参数槽位均可编辑。
- 自定义句式、任务类型与槽位值按可见顺序进入 `GenerationBrief.prompt`。
- 编辑不会破坏灰色槽位、任务类型或提交按钮。
- 原有默认句式保持不变；单测、类型检查、构建及浏览器交互通过。
