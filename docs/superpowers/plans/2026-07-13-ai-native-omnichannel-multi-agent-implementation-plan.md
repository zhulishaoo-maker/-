# AI Native 全域营销多智能体实施计划

## 1. 目标

以 [AI Native 全域营销自动化平台设计](../specs/2026-07-13-ai-native-omnichannel-marketing-design.md) 为唯一需求基线，将现有会场原型升级为“快速生成＋全链路活动”双入口产品，并逐步建立营销活动智能体、五个专业智能体、创意资产工厂、会场引擎、渠道执行和跨触点数据闭环。

## 2. 实施原则

- 先建确定性合同和执行底座，再开放智能体权限。
- 智能体只提交动作意图，不直接写线上资产、发布或修改交易数据。
- 京东大促品牌压板、搜索框、安全区和法务脚注采用确定性合成。
- 每次生成固定输出 4 张，支持单图重试。
- 多智能体通过版本化任务合同通信，不共享可变内部状态。
- 所有生成、审核、投放、实验、停流和回滚全量审计。

## 3. 目标模块

```text
apps/
  marketing-web/          # 快速生成、Campaign 工作台、专业智能体界面
  venue-web/              # 会场渲染与统一埋点
  api/                    # 领域 API、权限、审批和查询
  worker/                 # 生成、压板、发布、指标、实验与智能体任务
packages/
  prompt-composer/        # 句内槽位、默认句式、GenerationBrief
  agent-contracts/        # AgentTask、AgentResult、ActionIntent
  agent-runtime/          # 营销活动智能体与专业智能体
  asset-factory/          # 生图、图片处理、固定 4 张、资产衍生
  overlay-engine/         # 京东品牌/搜索框/资源位压板
  page-schema/            # 会场 Schema 与迁移
  component-registry/     # 会场组件库
  channel-adapters/       # 资源位、排期、投放、停流
  rules-engine/           # 硬规则、审批、频控、预算、授权
  event-contracts/        # Campaign 到 Order 统一标识
  experiments/            # ExperienceVariant、分流、判胜、回滚
```

## 4. 关键合同

### 4.1 GenerationBrief

包含任务类型、可编辑自然语言、句内槽位值、Campaign 继承值、输入 KV/商品、京东压板版本、风格、尺寸、安全区和生成状态。

### 4.2 OverlayTemplate

包含模板类型、版本、适用渠道/尺寸、固定层、可编辑文本层、安全区、禁放区、字体、色彩、资产引用和校验规则。

### 4.3 AgentTask / AgentResult

`AgentTask` 包含任务目标、业务上下文、允许工具、自治权限、成功标准、时间和成本上限。`AgentResult` 包含产出引用、证据、预期、风险、置信度和建议动作。

### 4.4 ActionIntent

统一表达生成、压板、审核、发布、实验、扩量、停流和回滚意图。每个意图必须通过权限、规则和幂等校验才能执行。

## 5. 实施阶段

### 阶段 0：保护现有原型与工程基线

1. 将当前未提交的会场原型作为独立基线提交。
2. 保持 Page Schema、会场组件和渲染器的模块边界。
3. 补充 CI、格式化、日志、错误边界和端到端测试骨架。

**验收：** 现有原型可重现，类型检查、单测和生产构建全绿。

### 阶段 1：双入口与句内 Prompt Composer

1. 将首页改为“快速生成＋全链路活动”。
2. 实现开屏、海报、Banner/资源位和会场默认句式。
3. 实现句内可编辑文本与下拉槽位，产出 `GenerationBrief`。
4. 支持 Campaign 继承、缺失项校验、上传 KV/商品图。
5. 固定生成数量为 4，不展示数量选项。

**验收：** 文本和槽位双向同步；不同任务输出合法 Brief；键盘和读屏器可完成所有操作。

### 阶段 2：京东压板与创意资产工厂

1. 定义 `OverlayTemplate` 及版本和预览。
2. 实现京东大促品牌压板和京东搜索框压板样板。
3. 实现底图生成任务，每任务创建 4 个候选槽位。
4. 实现确定性压板合成、文本适配、安全区和禁放区检测。
5. 实现单图重试、资产衍生关系、版本、审核和归档。

**验收：** 4 张候选独立状态可追踪；Logo、搜索框和 CTA 不由模型绘制；压板及安全区校验通过率达到试点门槛。

### 阶段 3：Campaign 与资产旅程工作台

1. 创建 Campaign、CampaignBrief、JourneyStage、MasterCreative、AssetVariant、Placement 和 VenueVersion。
2. 实现曝光、兴趣、到达、转化和再传播旅程图。
3. 支持快速生成资产升级为 Campaign 资产。
4. 串联创意、资源位、会场、审核和投放状态。

**验收：** 一个 Campaign 可一次管理至少 4 类资源和一个会场；所有资产可追溯至 Brief、底图、压板和审核。

### 阶段 4：确定性渠道与数据底座

1. 定义 Campaign→Journey→Placement→Creative→Venue→Order 标识。
2. 实现资源位适配器、排期、人群、频控、发布、停流和回滚。
3. 实现事件采集、去重、新鲜度、指标快照和数据质量告警。
4. 实现 ExperienceVariant、同期对照、样本门槛、最小效应量和护栏。

**验收：** 一次转化可追溯到创意、资源位和会场版本；数据异常时不判胜且自动化降级。

### 阶段 5：专业智能体

1. 实现 `AgentTask`、`AgentResult`、工具白名单、权限和审计。
2. 依次上线设计、会场运营、渠道运营、数据分析和治理智能体。
3. 为每个智能体建立独立离线评估集和成功指标。
4. 专业智能体只输出产出与 ActionIntent，不直接执行线上变更。

**验收：** 五个智能体的权限和工具集隔离；任务输入输出结构化；超时、重试和失败不会改变线上状态。

### 阶段 6：营销活动智能体编排

1. 实现目标拆解、并行委派、任务状态机和失败重规划。
2. 实现专业智能体冲突检测，以 Campaign 主目标和护栏裁决。
3. 实现人工接管、挂起、重试、取消和完整运行轨迹。
4. 实现 L0/L1，再逐步开放 L2 小流量实验。
5. 数据分析智能体提供证据，营销活动智能体决定扩量、停止或回滚。

**验收：** 营销活动智能体可并行调度专业智能体；所有线上动作经确定性规则执行；核心权益、预算和全量投放仍需人工审批。

### 阶段 7：试点与 L3 自治

1. 先回放历史活动，验证创意、会场、渠道和数据智能体。
2. 选择低风险活动开启 L0/L1，稳定后开启 L2。
3. 预注册“创意＋资源位＋会场”组合实验。
4. 连续稳定、回滚演练和风险评审通过后，仅对已验证策略开启 L3。

**验收：** 达到规格文档的 MVP 验收标准；否则保持低自治等级。

## 6. 必测场景

- 句内槽位与自然语言双向同步。
- 四个候选中部分生成失败后的独立重试。
- 京东品牌压板、搜索框、安全区、禁放区和文案溢出。
- 智能体提交超权、超预算或不合规 ActionIntent。
- 专业智能体输出冲突与营销活动智能体重规划。
- 智能体超时、重复委派、任务取消和人工接管。
- 渠道发布部分失败、数据延迟、埋点断链、频控/预算越界。
- 小流量实验判胜、扩量、停流、回滚和安全页。

## 7. 第一个代码迭代

1. 保护性提交现有会场原型。
2. 新建主首页，展示快速生成和全链路活动入口。
3. 实现开屏任务的句内 Prompt Composer。
4. 内置京东大促品牌压板、京东搜索框压板、风格和尺寸槽位。
5. 实现从槽位和文本产出 `GenerationBrief` 的类型与测试。
6. 将提交结果转为固定 4 张候选任务占位并展示状态。

## 8. 开工前外部依赖

- 京东大促品牌压板、搜索框压板和站内资源位的正式模板资产与版本规则。
- 可用字体、Logo、大促标识、法务脚注和授权边界。
- 生图、图片处理和素材存储服务。
- 商品、权益、人群、资源位、发布和数据平台接口。
- 试点活动、运营责任人、人工基线和风险审批人。
